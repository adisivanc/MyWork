Specificity in CSS is a way to determine which CSS rules will be applied to an element when there are multiple conflicting rules targeting the same element. It is used by the browser to determine the priority or specificity of CSS rules and resolve conflicts.

The specificity of a CSS selector is calculated based on the components of the selector and their corresponding values. The following factors contribute to the specificity value:

  1. Inline styles: Inline styles have the highest specificity because they are directly applied to the element using the style attribute. They are specified within the element's HTML tag and override any other CSS rules.

    1,0,0,0

  2. IDs: Selectors with ID attributes have higher specificity than classes and elements. An ID selector is denoted by the # symbol followed by the ID value.

    0,1,0,0

  3. Classes, attributes, and pseudo-classes: Selectors with classes, attributes, and pseudo-classes have a lower specificity than IDs. They are denoted by a dot (.) for classes, square brackets ([]) for attributes, or a colon (:) for pseudo-classes.

    0,0,1,0

  4. Elements and pseudo-elements: Selectors based on element names or pseudo-elements have the lowest specificity. They are denoted by the element name or the :: notation for pseudo-elements.

    0,0,0,1

The specificity is calculated by assigning a specific weight or value to each component of the selector. The more specific components a selector has, the higher the specificity value. In case of conflicts, the browser applies the rule with the highest specificity.

If two or more rules have the same specificity, the order in which they appear in the CSS file becomes the deciding factor. The last rule encountered by the browser will take precedence.

Understanding the concept of specificity is crucial for writing maintainable and predictable CSS code. It helps you control which styles are applied to specific elements and avoid unintended styling conflicts.
